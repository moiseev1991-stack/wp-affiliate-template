# Scaffold-site skill

Build a complete affiliate site from this template: theme, configs, 50 articles, infrastructure files. Optionally push to GitHub and deploy to Cloudflare.

## Inputs

**Required (ask if missing):**
- `domain` — full hostname, e.g. `example.pl`
- `language` — ISO 639-1 code, e.g. `pl`, `en`, `de`, `cs`, `sk`, `ru`
- `niche` — one of `casino`, `sports`, `crypto`, `health`, `design`, `generic`
- `money.url` — affiliate target URL
- `money.anchor` — brand name to use as link text
- `money.bonus` — bonus phrase, used as a tagline in money articles
- `donor` — reference site URL (for niche/style cues — what the new site is "based on")

**Optional (skip silently if missing):**
- `repo` — `owner/name` GitHub repo. Without this, scaffold runs locally and skips push.
- `GITHUB_TOKEN` env — without it (or `gh auth login`), no push or secret-set.
- `OPENAI_API_KEY` env — only needed for the **monthly cron** (one auto-post per month). Not used at scaffold time.
- `CLOUDFLARE_API_TOKEN` + `CLOUDFLARE_ACCOUNT_ID` env — without them, deploy is skipped.

## Step 1 — Resolve inputs

In order of preference:
1. Read `scaffold.config.json` if present.
2. Read `SCAFFOLD_*` env vars if `scaffold.config.json` is absent.
3. **Ask the user for any missing required field.** Use a single concise message listing all missing fields. Do NOT ask for optional fields.

If the user is on a remote/headless server and explicitly says "no questions" / "unattended", abort with a clear error listing missing fields rather than asking.

## Step 2 — Run the infra-only scaffold pre-pass

```
node scripts/scaffold.mjs
```

This pre-pass writes `lib/config.ts`, `app/globals.css` (randomized palette), `scripts/generation.config.json`, `package.json`, `robots.txt`, `wrangler.toml`. It also picks the deterministic monthly-post day from the domain seed and stores it in `lib/config.ts.monthlyPostDay` and `scripts/generation.config.json.monthlyPostDay`.

`scripts/scaffold.mjs` does **not** generate articles. It does not call OpenAI.

## Step 3 — Generate articles via Write tool

You (Claude) write all 50 articles directly to `content/posts/<slug>.mdx`. **Do not call OpenAI. Do not run `scripts/generate-post.mjs`.** That script is for the deployed cron only.

### Article distribution

- **5 money articles** — title format: brand review / brand bonus / brand mobile / brand VIP / brand payments (adapt phrasing to the language). Frontmatter: `featured: true`. The first paragraph must contain a markdown link `[Brand](money.url)`. Body: 1600+ words, 8 sections, dense ${'$'}{language} prose, **bold** key terms, concrete numbers. End with a localized editorial disclaimer in italics.
- **45 topical articles** — title and content drawn from the niche topic categories in `scripts/generation.config.json.topicCategories`. Frontmatter: `featured: false`. **Zero outbound brand links.** The brand name may appear as plain text but never as a link, never bold-without-link. The render-time `enforceMoneyLinkPolicy` in `lib/internal-links.ts` strips violations, but generate clean content from the start.

Spread `date:` values across the previous 6 months to look like organic backfill, not a single-day batch.

### Frontmatter shape

```yaml
---
title: "<title in target language>"
slug: "<lowercase-kebab>"
date: "YYYY-MM-DD"
description: "<SEO description, 155 chars max>"
emoji: "<one of 🎰 🪟 🪵 🎨 🌿 💻 ✨ 🍳 💡>"
featured: true | false
image: "/illustrations/<slug>.svg"
---
```

### SVG illustrations

For each article, write a small decorative SVG to `public/illustrations/<slug>.svg`. 320×180 viewBox, dark gradient bg, centered geometric focal element, no text/emoji. Reuse a single template with per-slug seed variation — they don't need to be unique masterpieces, just present so the article page hero block doesn't 404.

## Step 4 — Re-run the infra pass (now that articles exist)

```
node scripts/scaffold.mjs
```

The second pass picks up the 5 `featured: true` articles from disk, populates `siteConfig.moneyArticleSlugs`, and runs `npm install && npm run build` to verify everything compiles.

## Step 5 — Optional: push + deploy

If `cfg.repo` is set AND a GitHub token is available, the second-pass scaffold automatically:
- inits git
- pushes to `<owner>/<name>`
- sets `OPENAI_API_KEY` / `CLOUDFLARE_*` / `ANTHROPIC_API_KEY` repo secrets (only those present in env)
- runs `wrangler deploy` if Cloudflare creds are set

If any of these are missing, the relevant step is skipped with a `WARN:` line — no failure.

## Money-link policy (enforced at render time)

| Surface | Money links |
|---|---|
| Home page (10 cards) | 3 (slots 1, 4, 7 only) |
| 5 money articles | 1 each (first paragraph) |
| All other articles + categories + about/contact/privacy | 0 |
| **Total** | **8 links across 6 pages** |

`lib/internal-links.ts.enforceMoneyLinkPolicy` strips any extra money links the article generator may have inserted, and strips bold formatting on plain brand mentions outside the 5 money articles. You don't need to enforce this at write time, but cleaner is better.

## Per-site uniqueness (handled automatically)

`lib/uniqueness.ts` reads `siteConfig.themeSeed` (= the domain) and deterministically picks:
- **Menu preset** — brand alignment, nav alignment, categories as dropdown / flat / sidebar, search affordance, optional top-bar.
- **Layout preset** — sidebar position (left / right / none), home section order, optional tag cloud, 2 vs 3 column grid.
- **Emoji style** — plain / circle / badge / bordered / soft-shadow rendering.
- **WordPress fingerprint** — theme slug (neve, astra, generatepress, kadence, twentytwentyfour, oceanwp), WP version (6.3.x..6.7.x), 3-5 plugin signatures (yoast, contact-form-7, jetpack, wpforms, …) emitted as `<link>` tags in `<head>`.
- **CSS prefix** — class prefix used by Footer.

Two sites built from the same template thus differ in: HTML class names, header layout, sidebar position, home section order, emoji visual treatment, and WordPress fingerprint signals.

## What this skill does NOT do

- Does not create the GitHub repo. Create it on github.com first if you want auto-push.
- Does not configure DNS. Point DNS to Cloudflare manually after first deploy.
- Does not enable Cloudflare custom domain binding (one-shot dashboard op).
- Does not call OpenAI at scaffold time. OpenAI is only used by the deployed monthly cron.
