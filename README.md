# wp-affiliate-template

Reusable template for SEO affiliate blogs. Scaffolds a complete site (theme, configs, 50 articles, infra) from a small config file. Articles are generated by **Claude Code** at scaffold time — OpenAI is used **only** by the deployed monthly cron for one auto-post per month.

**Stack:** Next.js 14 · React 18 · TypeScript · Tailwind · MDX · GitHub Actions · Cloudflare Workers.

## What it ships with

- **Static Next.js site** with WordPress camouflage (`/wp-login.php`, `/xmlrpc.php`, `/wp-json/`, WP body classes, `<meta generator="WordPress …">`, theme & plugin `<link>` fingerprints).
- **Per-site uniqueness** (deterministic from domain seed):
  - 5 menu presets (brand alignment, nav alignment, categories as dropdown / flat row / sidebar link, optional top-bar, optional search box)
  - 5 layout presets (sidebar left / right / none, home section order, 2 vs 3 column grid, optional tag cloud)
  - 5 emoji-rendering styles (plain / circle / badge / bordered / soft-shadow — same emojis, different visual treatment)
  - 6 WP themes × 5 WP versions × 3-5 random plugin signatures
  - Per-site CSS class prefix
- **Flat URL structure** — articles at `/<slug>/`, no `/blog/` prefix.
- **Money-link policy** (hard-capped at render time): 8 links across 6 pages — 3 on home (slots 1/4/7) + 1 in first paragraph of each money article. Every other page: 0 money links, brand as plain text.
- 5 internal links auto-injected per article via `lib/internal-links.ts`.
- 9 emoji-themed categories.
- `robots.txt` open to all major search engines AND AI/LLM crawlers.
- **Monthly auto-post** — GitHub Actions workflow runs **daily**; only generates on the site's chosen day (1..28, deterministic per-domain), so different sites post on different days. ~$0.005 per article via OpenAI gpt-4o-mini.

## Inputs

**Required:**
- `domain` — `example.pl`
- `language` — ISO 639-1 (`pl`, `en`, `de`, `cs`, `sk`, `ru`)
- `niche` — `casino` | `sports` | `crypto` | `health` | `design` | `generic`
- `money.url` / `money.anchor` / `money.bonus` — affiliate target + brand + bonus phrase
- `donor` — reference site URL (used only for niche cues)

**Optional (all skip silently if absent):**
- `repo` — `owner/name` GitHub repo. Without it, scaffold runs locally only.
- `OPENAI_API_KEY` — only needed for the deployed monthly cron. Not used at scaffold time.
- `GITHUB_TOKEN` — enables auto-push.
- `CLOUDFLARE_API_TOKEN` + `CLOUDFLARE_ACCOUNT_ID` — enables auto-deploy.

## Pipeline

```
/scaffold-site (Claude skill)
   │
   ├─ 1. Resolve inputs (config or ask user for missing required fields)
   ├─ 2. node scripts/scaffold.mjs   (infra pre-pass)
   │       ├─ donor probe (optional)
   │       ├─ randomize :root palette
   │       ├─ pick monthly post day from seed (1..28)
   │       ├─ write lib/config.ts
   │       └─ write scripts/generation.config.json
   │
   ├─ 3. Claude writes 50 .mdx via Write tool
   │       ├─ 5 money articles (featured: true, 1 link in para 1)
   │       └─ 45 topical articles (featured: false, 0 outbound brand links)
   │
   └─ 4. node scripts/scaffold.mjs   (infra second-pass)
           ├─ pick up money slugs from disk
           ├─ npm install + build
           ├─ git push (if repo + token)
           ├─ gh secret set (only secrets present in env)
           └─ wrangler deploy (if CF creds)
```

## Structure

```
.claude/skills/scaffold-site/SKILL.md   ← scaffold flow, Claude-driven
.github/workflows/
  monthly-post.yml                      ← daily cron + day-of-month gate
  deploy.yml                            ← push-to-main → wrangler deploy
app/
  [slug]/page.tsx                       ← article page (sidebar L/R/none from preset)
  kategoria/[category]/page.tsx
  page.tsx                              ← home (10 cards, 3 money slots, layout preset-driven)
  layout.tsx                            ← WP fingerprint from uniqueness.ts
components/
  Header.tsx                            ← menu preset-driven
  Footer.tsx                            ← variation-aware
  Sidebar.tsx                           ← lang-aware, real category counts
  PostCard.tsx, MoneyBlock.tsx, Breadcrumbs.tsx
lib/
  config.ts                             ← siteConfig (incl. monthlyPostDay)
  uniqueness.ts                         ← per-site preset picks (seed = domain)
  categories.ts, money.ts, internal-links.ts, posts.ts
public/
  robots.txt                            ← search + AI bots, all allowed
  wp-login.php, wp-json/, xmlrpc.php    ← WordPress emulation
scripts/
  scaffold.mjs                          ← infra orchestrator (no OpenAI calls)
  generate-post.mjs                     ← MONTHLY cron generator (OpenAI, day-gated)
  generation.config.json                ← config for monthly cron
  randomize-theme.mjs                   ← per-site palette
  scrape-donor.mjs                      ← donor metadata probe
content/posts/                          ← Claude writes 50 .mdx files here
```

## Local development

```bash
npm install
npm run dev          # http://localhost:3000
npm run build        # static export to out/
```

## Usage (recommended — via Claude Code)

```bash
git clone https://github.com/moiseev1991-stack/wp-affiliate-template my-new-site
cd my-new-site

cp scaffold.config.example.json scaffold.config.json
# edit: domain, language, niche, money.{url,anchor,bonus}, donor, (optional) repo

claude /scaffold-site
```

The skill resolves missing required fields by asking once, then runs the full pipeline. Optional fields (`repo`, `OPENAI_API_KEY`, etc.) are silently skipped if absent.

## Usage (infra-only — without Claude)

```bash
node scripts/scaffold.mjs
```

This runs the infra pipeline (theme, configs, optional push/deploy) but **does not generate articles**. Useful for re-deployment after content changes, or as part of a custom orchestrator.
